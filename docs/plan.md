# Project Plan: Turnback Tape Audio

## 1. Risk Analysis (TAME)

| Risk | Impact | Strategy | Action Plan |
| :--- | :--- | :--- | :--- |
| **Responsive Design Issues** | Medium | **Mitigate** | Test the layout using the VS Code " Go Live" extension that listens on port 5500 throughout the build. |
| **Deployment Issues** | High | **Mitigate** | Upload all of my files to GitHub Pages early to double check a working URL before final submission. |
| **Time Management** | Medium | **Accept** | Realize that the project is due March 1st and prioritize the two-page requirement instead of other visual and functional features.  |
| **Scope Creep** | Low | **Eliminate** | I will avoid adding any JavaScript, animations, sounds, and interactive buttons as these are excluded from the current scope. |

## 2. Work Breakdown Structure (WBS)

### Phase 1: Planning and Documentation
* **Scope Finalization**: Complete the `scope.md` file.
* **Plan Development**: Draft the TAME risk analysis and WBS schedule (`plan.md`).
* **Repository Architecture**: Setup my GitHub repository and create the `/docs` folder.

### Phase 2: Technical Execution
* **HTML Structure**: Create `index.html` file with the sign-up form and `about.html` explaining what the business will entail.
* **CSS Theming**: Apply the Maxell inspired color palette (#D4B86A, #000000, #808080, #D22B2B) and retro Google Fonts (Orbitron, Montserrat) to `styles.css`.
* **Responsive Design**: First double check that the website is navigable on both mobile and desktop viewports.
* **Link Verification**: Test all internal links (`Home`, `About`) and the external link to my GitHub profile.

### Phase 3: Deployment and Reflection
* **Live Hosting**: Deploy to GitHub Pages to ensure the project is accessible via a public URL.
* **README Creation**: Develop a professional `README.md` that explains the project and links to my site and `/docs` folder.
* **Project Retrospective**: Complete the `retrospective.md` file answering what went well, what went wrong, and what I'd do differently next project.
* **Final Submission**: Submit the single GitHub repository link for grading.
